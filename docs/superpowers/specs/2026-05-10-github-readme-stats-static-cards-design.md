# GitHub Readme Stats Static Cards Design

## Context

This repository is the `JS-banana` GitHub profile repository. Its README currently embeds two images from the public `github-readme-stats.vercel.app` endpoint:

- `api/top-langs`
- `api`

Both URLs currently return `DEPLOYMENT_PAUSED`, so the README fails because the external public deployment is unavailable. The existing README workflow only updates WakaTime and blog sections; it does not generate stats cards.

## Goal

Replace the fragile public `github-readme-stats.vercel.app` image dependency with static SVG cards generated by GitHub Actions and stored in this repository.

## Approach

Use `readme-tools/github-readme-stats-action@v1`, the GitHub Actions path recommended by the upstream `github-readme-stats` documentation. Generate the two cards into `profile/` and update README to reference local SVG files:

- `profile/top-langs.svg`
- `profile/stats.svg`

Keep the existing WakaTime/blog workflow intact. Add a separate workflow for stats generation so each workflow keeps a single responsibility.

## Token Choice

Use the repository's existing `secrets.ACCESS_TOKEN` because the repository already relies on that secret in `.github/workflows/snk.yml`. If the secret is unavailable or lacks scope, the workflow can later be switched to `secrets.GITHUB_TOKEN` for public-only stats.

## Files

- Add `.github/workflows/readme-stats.yml` for static card generation and commit.
- Modify `README.md` to replace the two remote image URLs with local SVG references.
- Create `profile/.gitkeep` only if needed to keep the output directory visible before the first workflow run.

## Verification

- Validate workflow YAML syntax locally.
- Confirm README no longer references `github-readme-stats.vercel.app`.
- Run the new workflow manually with `gh workflow run` if available.
- Confirm `profile/stats.svg` and `profile/top-langs.svg` are generated by the workflow or note any GitHub-side permission/token issue clearly.
